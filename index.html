单例设计模式

发表于 2017-02-07 | 分类于 设计模式

单例模式的要点有三个：
1、单例模式类只能有一个实例
2、该类必须自行创建这个实例
3、该类必须对外提供这个实例
从具体实现角度来说，就是以下三点：
1、单例模式类的构造函数必须声明为私有
2、该类定义中含有一个该类的静态私有对象
3、该类提供了一个静态的公有的函数用于创建或获取它本身的静态私有对象。

饿汉式

懒汉式(较常用)

双重锁形式(比懒汉式性能更好)
单例模式中只需要一次创建实例的过程，双重锁形式只有第一次创建时才会执行同步代码块(即执行new Singleton()方法)，提高了效率(同步需要加锁，会消耗内存)

静态内部类(《Effective Java》中提到的方法)

枚举(可以防止反序列导致重新创建新对象)

Java中"=="和"equals"的使用

  • “==”
    • 对于基本数据类型用于比较两个数据的值是否相同
    • 对于引用数据类型用于比较两个对象的内存地址是否相同

  • “equals”

    • 基本数据类型不能用equals比较，但可以使用其对应包装类（即引用数据类型）来比较，且比较的是存放的值

使用Hexo+GitHub搭建博客

发表于 2017-01-05 | 分类于 Hexo

安装Hexo前提

您必须检查电脑中是否已安装下列应用程序：

Git
Node.js

Windows用户可从以上两个链接下载安装Git和Node.js

使用nvm安装Node.js(若已下载安装可直接跳过该步骤)

安装好Git后，新建一个文件夹作为根目录（文件名不要使用中文），在根目录下右键选择Git Bash，进入命令行终端，然后输入以下命令安装nvm
$ curl https://raw.github.com/creationix/nvm/master/install.sh | sh
又或者
$ wget -qO- https://raw.github.com/creationix/nvm/master/install.sh | sh
安装完成后，关掉命令行终端并用Git Bash重新打开，执行下列命令安装Node.js
$ nvm install stable

安装Hexo

安装好Git和Node.js后，新建文件夹（文件名不要使用中文），在文件夹内右键点选Git Bash(相关命令使用linux shell执行，以$开头的代码块都是在命令行终端执行)，进入命令行终端，执行以下命令安装Hexo
$ npm install -g hexo-cli

创建github仓库

新建一个名为的仓库，比如说，如果你的github用户名是test，那么你就新建的仓库（必须是你的用户名，其它名称无效），将来你的网站访问地址就是 http://test.github.io
